# Trademark Policy

## Summary

The Tula source code is released under the MIT license. You are free to use, modify, and distribute the software. The **Tula** name and the **RealActivity** name are not included in that license. You may fork the code, but you may not call your fork "Tula" or represent it as an official RealActivity product.

## Details

"Tula" and "RealActivity" are trademarks of RealActivity. The MIT license grants permission to use, modify, and distribute the *software*, but does not grant permission to use the Tula or RealActivity names, logos, or branding.

### Permitted Uses

- Fork the repository and use the code in your own project under a different name.
- State that your project is "based on Tula" or "built with Tula skills" for accurate attribution.
- Refer to Tula by name in articles, reviews, blog posts, academic publications, and discussions.
- Use the code commercially under your own brand.
- Contribute to the Tula project and receive contributor recognition.

### Restricted Uses

- Naming your product or service "Tula" or any confusingly similar variation.
- Using the Tula or RealActivity logos in your own product or service.
- Implying that your project is endorsed by, affiliated with, or an official version of Tula or RealActivity.
- Offering a hosted version of the software under the Tula name without written permission from RealActivity.

### Why This Policy Exists

Tula is a health-focused project. Users trust it with sensitive information about their health, their families, and their medical conditions. Trademark protection ensures that when individuals encounter the Tula name, they can have confidence in the origin and integrity of the project.

We want the code to be used as broadly as possible. We also want to ensure that the Tula name continues to represent a project maintained with genuine commitment to user privacy, data accuracy, and patient wellbeing.

### Questions

If you are uncertain whether your intended use of the Tula name is permitted, please open an issue or contact us directly. We are happy to discuss specific use cases.
